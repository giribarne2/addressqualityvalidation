TR Data Quality Demo - Validation Rules

# Rule Implementation
1 State Abbreviation Validates whether state name is formatted using two-letter abbreviation
2 Ordinal Number Abbreviation Validates whether input contains ordinal number word instead of preferred ordinal number figure (1st, 2nd, 3rd, etc.)
3 Business Word Abbreviation Validates whether input contains common presentations of business words as described by USPS standards seen here: Business Word Abbreviations
4 Zip Code Format Validates whether zip code is in Zip5-Zip4 format
5 Non-numeric character and domain validation Validates whether input contains symbols other than apostrophes, ampersand, hyphens, or domain names (.com, .net, .gov, .edu, .org)
6 Carriage return validation Validates whether input contains any carriage returns (tabs, new lines, returns)
7 Upper-case validation Validates whether input is upper case
8 Max length validation Validates whether input contains more than 100 characters
9 Non-latin character validation Validates whether input contains ASCII or Latin-1 Supplemental characters (see Unicode Charts)
10 Multiple spaces validation Validates whether input contains multiple spaces between words
11 Leading spaces validation Validates whether input contains leading spaces
12 Trailing spaces validation Validates whether input contains trailing spaces
